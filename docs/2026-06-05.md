# 2026-06-05 — Chia sẻ màn hình (screen share) qua Daily.co — Phase 1

## Mục tiêu phiên

App đã có canvas chung, chat, phiên dịch, gọi audio — **còn thiếu chia sẻ màn hình** (1 người Present → mọi người xem), tính năng cơ bản của mọi nền tảng họp. Phiên này **bàn plan kỹ → chọn hạ tầng → code Phase 1**.

## Quyết định hạ tầng (bàn kỹ trước khi code)

Khảo sát toàn cảnh build-vs-buy cho media (SFU vs CPaaS):

- **Self-host SFU (LiveKit/mediasoup/Janus) → LOẠI:** chủ dự án là vibe coder, không team IT → không nuôi server media.
- **Cloudflare Realtime → LOẠI cho giờ:** rẻ + hợp hạ tầng CF, nhưng **không có recording sẵn** → phải tự xây "recorder bot" + 1 server 24/7 (vài tuần kỹ sư) → quá sức.
- **Zoom Video SDK → LOẠI:** mạnh + phủ TQ, nhưng web SDK đòi **SharedArrayBuffer / cross-origin isolation** → vỡ phần load R2/font/IFC cross-origin của app. Tích hợp khó nhất.
- **Agora → không cần:** mạnh nhất cho Trung Quốc, nhưng phiên chốt "quên China đi".
- **→ CHỌN Daily.co (managed):** WebRTC thuần (không vướng cross-origin isolation), recording sẵn, DX sạch, **zero-ops** (không nuôi server — chỉ thêm 1 hàm token vào Worker). Giá ~$0.0015–0.004/người/phút, free 10k phút/tháng.

Kiến trúc chốt: **Daily = ống dẫn media; socket Collab sẵn có = presence + single-share lock; lazy-join** (chỉ nối Daily khi có người share → tiết kiệm phút). Token mint server-side, `DAILY_API_KEY` chỉ Worker thấy (ở `worker/.dev.vars`, đã gitignore).

## Đã làm hôm nay (Phase 1 — typecheck pass 0 lỗi, app boot 0 console error)

Lập **team agent đọc toàn cảnh** (audio subsystem làm template, UI mount points, daily-js@0.90.0 API thật) trước khi code để khớp chuẩn, không patch mò.

1. **Worker token endpoint** — `worker/src/index.ts`: `GET /v1/daily/token?roomId=` → ensure Daily room (GET, tạo on-404, private) + mint meeting-token (`canSend:["screenVideo","screenAudio"]`, hết hạn 4h) → trả `{url, token}`. **Test curl ra token thật** (xác nhận body meeting-tokens phải nested trong `properties` — doc-summary của model fast nói sai top-level).
2. **Client** `getDailyToken()` — `data/projects.ts`, mirror `getMeeting` (cùng `STORAGE_URL`).
3. **Presence + single-share lock** (mirror RAISE_HAND, mã hoá tự động): `WS_SUBTYPES.SCREEN_SHARE` (app_constants), type (data/index), `Portal.broadcastScreenShare`, `Collab`: `screenShareStateAtom` (Map socketId→true) + handler + `applyScreenShare` + `setScreenShare` (CollabAPI) + **dọn presence khi peer rời** (mở khoá nếu sharer rớt).
4. **screenshare/** (mirror audio/): `screenShareState.ts` (atoms media + instance), `DailyScreenShare.ts` (call-object: ensureJoined `videoSource:false/audioSource:false` → join → startScreenShare; track-started `screenVideo` → MediaStream cho viewer; lazy leave), `ScreenShareController.tsx` (lifecycle + **lazy join/leave theo presence**), `ScreenSharePane.tsx` (cửa sổ nổi + **Pop-out Document-PiP** ra màn hình 2, fallback `window.open`), `popOut.ts`, `screenshare.scss`.
5. **Wire UI**: nút **Present** (header) + active/disabled state + **lock** ("X đang trình bày") + **viewOnly gate** (ẩn khi review); mount pane + controller ở MeetingShell; **badge 📺** ParticipantsBar; **i18n en/vi/ko** (block `screenShare`).

## Cơ chế chốt (chống lỗi)

- Single-share lock: presence atom — ai share trước chiếm slot; người khác bấm Present bị **disable + tooltip "đợi finish"** (chưa cần host-approve — chốt bỏ).
- Mã hoá broadcast tự động (y raise-hand). Peer rời → tự mở khoá. Socket-ready guard trước khi join Daily. viewOnly (cuộc họp đã xong) → ẩn toàn bộ.
- Pop-out dùng `<video>` PHỤ trong cửa sổ PiP (cùng MediaStream) thay vì dời node React → tránh vỡ reconcile.
