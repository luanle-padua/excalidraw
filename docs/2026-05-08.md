# 2026-05-08 — Meeting Canvas: realtime collab setup qua Cloudflare Tunnel

## Mục tiêu phiên

Test Excalidraw realtime collab giữa **desktop** (máy bạn) + **labmate trên LAN** + **iPhone trên WiFi**, để chuẩn bị làm tool meeting nội bộ (canvas, share file/ảnh, mark, note, export).

## Kết quả

3 thiết bị join cùng room, broadcast cursor + edit realtime qua tunnel. Setup hiện tại:

```
[browser desktop] ─┐
[browser labmate] ─┼─→ https://<random>.trycloudflare.com  (Cloudflare Tunnel)
[browser iPhone]  ─┘                  ↓
                          Vite dev :3001 (proxy /socket.io)
                                      ↓
                          room :3002 (socket.io broadcast server)
```

## 3-terminal workflow

(các command chạy từ root của repo CanvasMeet)

```powershell
# 1) room (socket.io broadcast server)
cd room
yarn start:dev

# 2) Vite dev (frontend + /socket.io proxy)
cd excalidraw-app
yarn start

# 3) Cloudflare quick tunnel (chạy từ repo root)
.\start-tunnel.ps1
# → in URL https://<random>.trycloudflare.com (share cho team)
```

URL tunnel **random mỗi lần** restart cloudflared. Để URL ổn định: cần Cloudflare account + domain → named tunnel.

## Gotchas — quan trọng nhớ

### 1. Vite empty env KHÔNG override

`.env.local` ghi `VITE_APP_WS_SERVER_URL=` (rỗng) sẽ bị **ignore**, Vite fallback về giá trị trong `.env.development` (`http://localhost:3002`). Trên desktop OK vì localhost = chính máy đó. Trên phone, `localhost` = chính phone → fail.

**Fix:** dùng cờ phụ (`VITE_DEV_TUNNEL=true`) — giá trị non-empty nên override được — rồi check trong code:

```ts
const tunnelMode = import.meta.env.VITE_DEV_TUNNEL === "true";
const wsServerUrl = tunnelMode ? "" : import.meta.env.VITE_APP_WS_SERVER_URL;
```

Cùng lý do: `VITE_APP_FIREBASE_CONFIG=` (rỗng) cũng không override → set `={}` (object rỗng — non-empty string).

### 2. socket.io-client không nhận URL `undefined`

TypeScript fail với `socketIOClient(undefined, opts)`. Phải split:
```ts
const socket = wsServerUrl
  ? socketIOClient(wsServerUrl, wsOptions)
  : socketIOClient(wsOptions);
```
URL trống → socket.io-client tự dùng `window.location.origin`.

### 3. Firebase OSS dev project reject ghi từ origin lạ

Excalidraw mặc định dùng Firebase project `excalidraw-oss-dev` (project public của OSS team). Project này có security rules — ghi từ origin không nằm trong allowlist (như `*.trycloudflare.com`) trả `failed-precondition` 400.

**Fix:** đã guard trong `firebase.ts` — khi `FIREBASE_CONFIG.projectId` rỗng, mọi function no-op, app vẫn realtime qua socket.io. Mất tính năng persist scene snapshot lên cloud — chấp nhận được cho meeting nội bộ.

Nếu muốn persist sau này: tạo Firebase project riêng, paste config vào `VITE_APP_FIREBASE_CONFIG` của `.env.local`.

### 4. iOS Safari debug trên Windows: dùng Eruda

Không có Safari Web Inspector trên Windows. Eruda là in-page DevTools cho mobile, chỉ 1 script tag từ jsDelivr. Đã auto-load trong `index.html` cho dev hosts. Tap nút floating gear góc dưới để mở.

### 5. Service Worker stale trên máy đã từng test cũ

Trên máy bạn nếu có lúc dùng PWA build, service worker có thể giữ bundle cũ. Khi đổi config (như WS URL), cần:
- F12 → Application → Service Workers → **Unregister**
- Storage → **Clear site data**
- Đóng tab, mở lại

Trong dev mode mặc định, PWA SW chỉ được kích khi `VITE_APP_ENABLE_PWA=true` — bình thường không có. Nhưng vẫn nên check trên máy đã test với config khác trước đó.

### 6. Cert tự ký cho LAN: dead end (đã rollback)

Hướng đầu tiên tôi thử: `mkcert` + cert LAN cho cả Vite (HTTPS port 3001) và room (HTTPS port 3002), test labmate qua IP. Vấn đề:
- Browser **silent-fail WSS** khi cert không trust (không hiện prompt như HTTPS page)
- Phải accept cert per-port (page port + WS port riêng biệt)
- Mỗi máy mới cần cài root CA mkcert

→ Phức tạp, đã bỏ. Tunnel HTTPS thật đơn giản hơn nhiều.

## Decisions

- **Excalidraw thay vì tldraw**: tldraw có template `sync-cloudflare` rất đẹp (Cloudflare Workers + Durable Objects, sync built-in, R2 asset, URL preview), nhưng license tldraw phức tạp ("Production Environment" có thể tính cho internal tool — phải hỏi sales). Excalidraw MIT thuần — an toàn cho tool nội bộ công ty. → giữ Excalidraw.
- **Cloudflare quick tunnel thay vì named tunnel**: chưa cần URL ổn định, test trước.
- **Disable Firebase**: dùng socket.io làm primary realtime + LocalData (IndexedDB) làm offline cache. Không persist scene lên cloud.
- **Không port sang Durable Objects**: refactor lớn (excalidraw client dùng socket.io, DO chỉ raw WS — phải đổi cả client). Để sau khi product validate.

1. **URL ổn định** — mua domain → Cloudflare Registrar → named tunnel + Cloudflare Access (SSO email công ty, free 50 users)

6. **Room management** — auth, permissions, danh sách phòng, lịch họp

## Tham khảo nhanh

- Hash format: `#room=<roomId>,<roomKey>` — roomKey là AES-GCM key cho E2E encryption client-side. Server chỉ relay binary, không decrypt.
- Server protocol: `join-room`, `server-broadcast`, `client-broadcast`, `server-volatile-broadcast`, `room-user-change`, `user-follow`. Thấy ở [room/src/index.ts](../room/src/index.ts).
- Realtime sync handler client: [excalidraw-app/collab/Collab.tsx](../excalidraw-app/collab/Collab.tsx) (~900 lines).
