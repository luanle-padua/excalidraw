# Decoration assets

These PNG files are loaded as overlays on top of locked canvas images by
`excalidraw-app/components/mcm/PinnedImagesOverlay.tsx`. The overlay
picks one deterministically based on the file id hash, so each pinned
image looks slightly different but stays stable across reloads + peers.

## Folder structure

```
public/decorations/
├── tape/        Washi-tape strips. Wide aspect ratio (~3:1). Transparent
│                background, semi-transparent so the image shows through
│                faintly. Naming: tape-01.png, tape-02.png, …
├── stickers/    Cute decorative stickers (animals, plants, hearts, etc).
│                Roughly square (~1:1) with transparent background.
│                Naming: sticker-01.png, sticker-02.png, …
└── stamps/      Rubber-stamp / approval marks ("APPROVED", "URGENT").
                 Wider rectangular (~5:2). Semi-transparent ink texture.
                 Naming: stamp-01.png, stamp-02.png, …
```

## Asset specs

- **Format**: PNG with alpha channel (PNG-24 + alpha).
- **Resolution**: long edge ≈ 512 px (high DPI ready, file ≤ 80 KB).
- **Background**: must be transparent.
- **Quantity per kind**: 8–16 variants — enough variety without bloating
  the bundle. Too few and every other image looks identical.

## Where to source

- **Tape**: search "washi tape PNG transparent" on Pexels, Pixabay, or
  buy a pack from Creative Market ($5–10 typically). Free options exist
  on freepik but check the commercial-use licence.
- **Stickers**: OpenMoji (CC BY-SA 4.0) at https://openmoji.org/ has
  thousands of unified-style stickers; download the PNG variants
  individually. Twemoji is also free.
- **Stamps**: searchable on Freepik / Vecteezy, or generate via the
  `mcm-stamp-css` mode (a CSS-only fallback we build inline — see the
  overlay component).

## After adding files

The overlay component loads the manifest at
`public/decorations/manifest.json` (generated by the build), or scans
the folder if running in dev. To register new files manually, update:

```ts
// excalidraw-app/components/mcm/decorations/assets.ts
export const TAPE_ASSETS = [
  "/decorations/tape/tape-01.png",
  "/decorations/tape/tape-02.png",
  // …
];
```

The hash-based picker in `PinnedImagesOverlay` will then start using
them automatically.
